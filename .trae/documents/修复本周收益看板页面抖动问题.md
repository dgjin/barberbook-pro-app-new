## 问题原因

`useWeeklyStats` hook 中使用了 `new Date()` 作为依赖项，每次组件渲染都会创建新的 Date 对象，导致：

1. `fetchStats` 函数重新创建
2. `useEffect` 触发重新获取数据
3. `setStats` 更新状态导致重新渲染
4. 形成无限循环，造成页面抖动

## 修复方案

修改 `useWeeklyStats` hook，使用稳定的周起始日期字符串作为依赖，而非 Date 对象：

**修改文件**: `/Users/dgjin/kimi/barberbook-pro-app/hooks/useAdmin.ts`

**具体修改**:

1. 计算本周一的日期字符串（如 "2024-02-19"）作为稳定依赖
2. 用这个字符串替代 `currentDate` 作为 `fetchStats` 和 `useEffect` 的依赖
3. 同一周内依赖值不变，不会触发重复获取

**代码变更**:

```typescript
// 修改前
const currentDate = date || new Date();
// ... 依赖 [barbers, currentDate]

// 修改后  
const currentDate = date || new Date();
// 计算本周一的日期字符串作为稳定依赖
const dayOfWeek = currentDate.getDay();
const diffToMonday = dayOfWeek === 0 ? -6 : 1 - dayOfWeek;
const monday = new Date(currentDate);
monday.setDate(currentDate.getDate() + diffToMonday);
const weekKey = monday.toISOString().split('T')[0]; // "2024-02-19"
// ... 依赖 [barbers, weekKey]
```
